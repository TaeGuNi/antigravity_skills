# 🚀 High-Performance Data Pipeline & Minification Protocol

이 프로토콜은 AI 에이전트가 코드를 짤 때 "단순히 동작하는 수준"을 넘어, 타겟 시스템(브라우저 봇, API 소비자 등)을 고려한 **가장 완벽한 아키텍처적 성능과 메모리/대역폭 효율성**을 달성하기 위한 절대 원칙입니다.

---

## ⚖️ 제 1원칙: M2M(Machine-to-Machine) 미니피케이션 강제

**사람이 읽는 화면이 아닌 기계(크롤러 봇, API 파서)가 읽는 데이터 포맷을 렌더링할 때, 심미적인 띄어쓰기나 줄바꿈을 코드에 삽입하는 아마추어 같은 행동을 엄격히 금지한다.**

1. **대상 포맷:** XML (Sitemap, RSS), JSON (API Response), HTML (SSR 메타 태그 등)
2. **규칙:**
   - 백틱(Template Literal) 내부에 가독성이나 시각적 구분을 위한 **불필요한 공백, 탭, 줄바꿈(`\n`) 사용을 절대 금지**한다.
   - 모든 문자열은 가장 완벽하게 압축된 한 줄 짜리 덩어리(Minified Block)로 생성되어야 한다.
   - *이유:* 대용량 렌더링 시 스페이스바 하나가 누적되어 엄청난 오버헤드와 네트워크 대역폭 비용(Bandwidth Cost)을 유발하기 때문이다.
3. **가독성 확보 방법:** 반환값(XML)에 공백을 넣는 대신, 반환값을 만들어내는 **TypeScript 템플릿 코드 바깥에 주석과 헬퍼 함수를 적용**하여 개발자의 코드 가독성을 챙긴다.

---

## 🧮 제 2원칙: 다차원 반복(O(N^2)) 금지 및 Source of Truth 탐색

**거대한 배열(Array) 데이터를 다룰 때, 직관적인 중복 루프(`forEach` 안의 `map` 등)를 돌기 전 반드시 "데이터의 최소 중심축"을 먼저 찾아야 한다.**

1. **규칙:**
   - 7,000개가 넘는 노드를 그릴 때, 각 노드 안에서 또 다른 10개의 속성 배열을 돌면(`7,000 * 10 = 70,000`) 프론트 서버의 메모리(GC)는 폭발한다.
   - 루프를 개시하기 전, "**가장 본질적인 단 하나의 원본(Source of Truth)**"이 무엇인지 파악하고 메인 루프는 오로지 **최소 단위**에서만 실행되어야 한다.
   - *예시:* 다국어 사이트맵의 본질은 "10개의 언어가 아니라, 1개의 콘텐츠 베이스라인과 10개의 부가 맵핑(Alternate Links)"이다. 따라서 메인 노드(`url`)는 배열의 1원소 기준(`DEFAULT_LANGUAGE`)으로 필터링한 1번만 찍어내고, 파생 데이터(`hreflang`)는 그 하위에서 곁다리로 삽입한다.

---

## 🗑️ 제 3원칙: 무의미한 임시 메모리(Array) 할당 차단

**루프나 체이닝 파이프라인에서 직관적인 1회성 배열 조작을 위한 무명 배열 생성(`[]`)을 지양한다.**

1. **규칙:**
   - 로직이 상수 1개를 대상으로 확정(Deterministic)된 경우, `[DEFAULT_LANGUAGE].map()` 처럼 쓸데없는 길이 1짜리 배열 인스턴스를 만들고 돌리는 낭비를 하지 마라.
   - 파라미터가 1개라면 해당 데이터를 배열 스코프가 아니라 **직접 타겟 함수(Helper)**로 꽂아 넣는다.
   - *이유:* 배열 메서드를 남용하는 선언적 프로그래밍의 맹점은 내부적으로 끊임없이 중간 가비지(Garbage Collection)가 생겨 메모리 스파이크를 만든다는 점이다.

---

## 🧩 제 4원칙: 로직(Logic)과 렌더링(Rendering)의 캡슐화 분리

**아무리 짧은 반복문이라도 상태 감별 로직(if-else, SEO 가중치 산출)과 렌더링 문자열 조합이 한 블록 안에 섞여 들어가는(Spaghetti) 꼴을 허용하지 않는다.**

1. **규칙:**
   - HTML/XML 태그 리터럴 코드가 위치한 클로저(Closure) 함수 안에는 **계산 로직이 단 한 줄도 없어야 한다.**
   - 복잡한 수식이 필요한 경우(예: 날짜를 일수로 치환하여 Priority를 부여), 반드시 외부의 순수 함수(`Pure Function`)로 적출해야 한다. (e.g. `getSeoProperties()`)
   - 렌더러 함수(e.g. `createXmlUrlBlock()`)는 오직 **"계산이 끝난 퓨어 데이터"**를 받아서 한 줄의 텍스트 조합으로 반환하는 단일 책임(SRP)만을 가진다.
